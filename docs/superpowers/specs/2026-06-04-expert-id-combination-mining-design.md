# 提取符合条件的专家ID - 专家ID组合筛选设计文档

## 目标

调整现有 Windows 桌面程序“提取符合条件的专家ID”的核心逻辑。

旧逻辑是按单个期次列统计某个专家 ID 的出现次数。新版逻辑改为查找“专家 ID 组合”：一组专家 ID 只要作为同一个集合共同出现在多个期次列中，并满足最少组合人数和最少出现期次数，就输出到结果 sheet。

## 输入文件

继续支持：

- `.xlsx`
- `.xls`

用户仍通过界面选择输入文件和 sheet。

期次识别规则沿用旧版：

- 从上到下扫描所选 sheet；
- 第一行中只要存在 `6-8 位纯数字` 单元格，就将该行识别为期次行；
- 该行中所有 `6-8 位纯数字` 的列作为期次列；
- 如果期次行下一行是 `1、2、3...` 这类辅助行，则真正专家 ID 数据从再下一行开始；
- 否则专家 ID 数据从期次行下一行开始。

## 专家 ID 清洗规则

每个单元格作为一个专家 ID 候选值处理：

- 转为字符串；
- 去掉前后空格；
- 清理后为空则忽略；
- 不删除中间空格；
- 不做大小写转换；
- 不做繁简转换；
- 不做全角半角转换。

同一个期次列中，如果同一个专家 ID 出现多次，只保留一次。

## 组合定义

每个期次列会形成一个去重后的专家 ID 集合。

从每个期次集合中枚举所有专家 ID 组合，组合人数必须 `>= 每组最少ID数`。

组合相同的判断标准：

- 只看包含的专家 ID 是否完全一致；
- 不看顺序；
- 去掉前后空格后的 ID 完全一致才算同一个 ID；
- `A，B，C，D，E` 和 `E，D，C，B，A` 算同一组；
- `A，B，C，D，E` 和 `A，B，C，D，E，F` 不算同一组。

大组合和子组合如果都满足条件，则全部输出。

例如某期次有 `A，B，C，D，E，F`，则会产生所有人数不少于阈值的组合。若 `A，B，C，D，E` 与 `A，B，C，D，E，F` 都满足出现期次数要求，两组都输出。

## 参数

界面新增两个下拉参数：

- `每组最少ID数`
  - 范围：`2-100`
  - 默认值：`5`
- `最少出现期次数`
  - 范围：`2-100`
  - 默认值：`4`

只有同时满足以下条件的组合才输出：

- 组合人数 `>= 每组最少ID数`
- 出现期次数 `>= 最少出现期次数`

## 出现期次数

出现期次数按“期次列数”计算，不按期次编号去重。

如果两个期次列的列头都为 `2024147`，且同一组合都出现在这两列中，则出现期次数计为 `2`。

结果中的具体期次也按列顺序重复写出，例如：

`2024147，2024147`

## 算法设计

不能直接暴力枚举每个期次列中所有组合。

按当前样例文件测算：

- 期次列数：`156`
- 每个期次去重后专家 ID 数最大：`67`
- 平均去重后专家 ID 数约：`44.04`
- 若直接枚举每列所有 `5人及以上` 组合，理论组合数约为 `240,668,014,983,629,197,022`

直接枚举不可行。

采用频繁项集/期次交集剪枝算法：

1. 将每个期次列转换为去重后的专家 ID 集合。
2. 丢弃去重后 ID 数少于 `每组最少ID数` 的期次列。
3. 为每个专家 ID 建立出现期次列集合。
4. 丢弃单个 ID 出现期次数少于 `最少出现期次数` 的 ID。
5. 从可用 ID 递归扩展组合。
6. 每扩展一个 ID，就对组合内所有 ID 的出现期次列集合求交集。
7. 如果交集大小小于 `最少出现期次数`，该组合及其更大扩展直接剪枝。
8. 如果组合人数达到 `每组最少ID数`，且共同出现期次数达标，则记录结果。
9. 已记录或已访问的组合使用排序后的 ID tuple 作为规范 key，避免重复处理。

该算法与“枚举全部组合后统计”的结果等价，但会提前剪掉不可能达标的组合。

按当前样例数据和默认参数试算，符合条件的组合约为：

- 总计：`931` 组
- `5人组合`：`798` 组
- `6人组合`：`123` 组
- `7人组合`：`10` 组
- `8人及以上`：`0` 组

## 输出顺序

结果按组合第一次被发现的顺序输出。

顺序定义为：

1. 按原 Excel 中期次列从左到右扫描；
2. 每个期次内按专家 ID 原始出现顺序构造候选；
3. 某个组合第一次被记录时的位置，就是最终输出顺序；
4. 后续再次遇到同一组合，只追加出现期次，不改变输出顺序；
5. 算法内部即使使用排序 tuple 做去重 key，也不能改变展示顺序。

每组专家 ID 展示顺序：

- 按该组合第一次被记录时的展示顺序；
- 如果组合是从大集合中枚举出来的子组合，则沿用该期次原始列中这些 ID 的相对顺序。

## 输出文件

输出为新的 `.xlsx` 文件，不直接覆盖输入文件。

默认输出文件名：

`原文件名_组合筛选结果.xlsx`

用户可以通过界面选择输出位置和输出文件名。

如果输出文件已存在，直接覆盖。

为保护原始数据，程序不允许输出路径与输入路径完全相同。如果相同，提示用户更换输出文件名。

## 工作簿复制规则

`.xlsx` 输入：

- 先复制原文件到输出路径；
- 在复制后的 workbook 中新增结果 sheet；
- 尽量保留原 workbook 内容，包括公式；
- 不承诺复杂 Excel 对象 100% 无损，例如宏、复杂图表、数据透视表、外部链接等。

`.xls` 输入：

- 输出为新的 `.xlsx`；
- 复制原工作簿中所有 sheet 的 sheet 名和单元格值；
- 不保留公式、样式、合并单元格、图表等。

## 结果 Sheet

新增结果 sheet。

sheet 命名规则：

- 优先使用 `结果`
- 如果已有 `结果`，使用 `结果1`
- 如果已有 `结果1`，使用 `结果2`
- 依次递增

判断依据是输出 workbook 中已有的 sheet 名。

结果 sheet 四列：

```text
专家ID组合 | 组合人数 | 出现期次数 | 出现期次
```

字段说明：

- `专家ID组合`：同一组合内专家 ID 用中文逗号 `，` 拼接；
- `组合人数`：该组合包含的专家 ID 数量；
- `出现期次数`：该组合共同出现过的期次列数；
- `出现期次`：具体期次编号，按原 Excel 期次列从左到右顺序，用中文逗号 `，` 拼接。

## 界面调整

程序名继续沿用：

`提取符合条件的专家ID`

界面包含：

- 输入文件选择；
- Sheet 下拉框；
- `每组最少ID数` 下拉框，范围 `2-100`，默认 `5`；
- `最少出现期次数` 下拉框，范围 `2-100`，默认 `4`；
- 输出文件选择，默认 `原文件名_组合筛选结果.xlsx`；
- 运行按钮；
- 进度条；
- 日志框。

日志需要展示：

- 输入文件；
- 选择 sheet；
- 期次行；
- 数据起始行；
- 识别期次列数；
- 有效期次列数；
- 不同专家 ID 总数；
- 参数值；
- 搜索到的结果组数；
- 输出文件路径；
- 明确失败原因。

如果参数较低，可能产生大量组合，日志中需要提示可能耗时较长。

## 错误处理

需要明确提示以下错误：

- 输入文件为空；
- 输入文件不存在；
- 输入文件格式不是 `.xlsx` 或 `.xls`；
- sheet 不存在或读取失败；
- 未识别到期次行；
- 未识别到有效期次列；
- 没有任何期次列满足最少 ID 数；
- 输出路径为空；
- 输出路径与输入路径完全相同；
- 输出目录不存在；
- 输出文件写入失败；
- 输出文件可能被 Excel 或 WPS 占用。

## 验收标准

用户可选择 `.xlsx` 或 `.xls` 文件。

用户可选择 sheet。

用户可设置：

- 每组最少 ID 数；
- 最少出现期次数。

程序能根据期次列生成专家 ID 集合，并找出所有满足条件的专家 ID 组合。

组合判断只看 ID 集合，不看顺序。

大组合和子组合如果都满足条件，需要全部输出。

结果写入新的 `.xlsx` 文件，并新增 `结果` sheet 或递增命名的结果 sheet。

结果 sheet 包含四列：

```text
专家ID组合 | 组合人数 | 出现期次数 | 出现期次
```

输出文件不能与输入文件同路径。

`.xlsx` 输入时，输出文件尽量保留原 workbook 公式和内容。

`.xls` 输入时，输出文件保留 sheet 名和单元格值。
