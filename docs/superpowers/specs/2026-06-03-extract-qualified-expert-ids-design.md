# 提取符合条件的专家ID - 设计文档

## 目标

构建一个 Windows 单文件 exe 程序，程序名为“提取符合条件的专家ID”。

程序从 Excel 文件读取数据，支持 `.xlsx` 和 `.xls` 输入。用户选择输入文件、sheet、阈值 `n`、输出位置和输出文件名后，程序按期次列统计专家 ID 出现次数，并将每个期次下出现次数 `>= n` 的专家 ID 输出到新的 `.xlsx` 结果文件。

## 已确认样例事实

样例文件实际位于：

`C:\Users\ZYB\Desktop\数据源.xlsx`

该文件当前读取到 1 个 sheet：`Sheet1`。

样例结构为：

- 第 1 行：仅 A 列有 `空白`
- 第 2 行：从 B 列开始是期次，例如 `2024147`
- 第 3 行：包含 `日期`、`1`、`2`、`3` 等辅助信息
- 第 4 行开始：专家 ID/名称数据

用户要求输出保留原始结构，但不要求保留字体、颜色、边框、列宽、合并单元格等视觉样式。

## 界面设计

使用简单 Windows 桌面窗口，包含以下控件：

- 输入文件选择：支持 `.xlsx`、`.xls`
- Sheet 下拉框：选择输入文件后读取 sheet 列表
- 阈值 `n` 下拉框：范围 `2-100`，默认值为 `4`，选项按从小到大展示
- 输出文件选择：允许用户选择输出目录和输出文件名
- 默认输出文件名：`原文件名_筛选结果.xlsx`
- 运行按钮
- 进度条
- 日志框

日志框需要展示关键处理步骤和详细错误原因，例如：

- 输入文件不存在
- Excel 文件无法读取
- Sheet 读取失败
- 未识别到期次行
- 未识别到期次列
- 输出文件无法写入
- 输出文件可能被占用
- 处理完成后的输出路径

## 技术方案

推荐使用：

- `Tkinter`：构建 GUI，减少额外依赖和打包体积
- `openpyxl`：读取 `.xlsx`，输出 `.xlsx`
- `xlrd`：读取 `.xls`
- `PyInstaller --onefile`：打包为单个 exe

不采用的方案：

- Excel COM 自动化：依赖本机安装 Microsoft Excel，环境不稳定
- PyQt/PySide：界面能力更强，但打包体积较大，不符合尽可能小的要求

## 期次识别规则

从上到下扫描所选 sheet。

遇到第一行中存在 `6-8 位纯数字` 单元格时，将该行判定为期次行。

在该期次行中，所有满足 `6-8 位纯数字` 的单元格所在列，作为需要处理的期次列。

该规则支持只有 1 个期次列的文件。

非期次列保留原始结构内容，但不参与专家 ID 统计。

## 专家 ID 统计规则

对每个期次列独立处理。

专家 ID 数据从期次行下方开始读取。样例中的辅助行如 `日期`、`1`、`2`、`3` 即使进入统计，通常也会因出现次数不足而被过滤。

每个单元格作为一个专家 ID 候选值处理：

- 转为字符串
- 去除前后空格
- 清理后为空则忽略

统计每个专家 ID 在当前期次列中的出现次数。

若出现次数 `>= n`，该专家 ID 符合条件。

每个符合条件的专家 ID 在输出中只保留一次。

输出顺序按该 ID 在原始期次列中第一次出现的顺序。

## 输出规则

输出文件格式固定为 `.xlsx`。

输出保留原始表格结构：

- 保留期次行
- 保留非期次列中的原始结构内容
- 保留样例中类似第 1 行和第 3 行的结构内容
- 保留所有期次列，即使某个期次没有符合条件的专家 ID

每个期次列下，符合条件的专家 ID 从数据区起始行开始连续输出，不保留原始行号空洞。

若某期次无符合条件的专家 ID，该期次列仍保留，数据区为空。

## 错误处理

程序应捕获并在日志框中展示明确错误信息。

主要错误类型：

- 输入路径为空
- 输入文件不存在
- 输入文件扩展名不是 `.xlsx` 或 `.xls`
- 无法读取 workbook
- 所选 sheet 不存在或读取失败
- 未找到包含 `6-8 位纯数字` 的期次行
- 找到期次行但没有有效期次列
- 输出路径为空
- 输出目录不存在
- 输出文件写入失败
- 输出文件被 Excel 或 WPS 占用

## 打包要求

最终交付一个单独的 Windows exe 文件。

打包目标是尽可能控制体积，但优先保证：

- 不依赖用户安装 Python
- 不依赖用户安装 Microsoft Excel
- 能读取 `.xlsx` 和 `.xls`
- 能输出 `.xlsx`

## 验收标准

程序可在 Windows 环境中启动。

用户可选择 Excel 文件，随后选择对应 sheet。

阈值 `n` 下拉框显示 `2-100`，默认 `4`。

用户可选择输出目录和输出文件名。

点击运行后，程序读取所选 sheet，识别期次行和期次列。

对每个期次列统计专家 ID，筛选出现次数 `>= n` 的 ID。

结果文件保持原始横向期次结构，每个符合条件的专家 ID 在对应期次列下只出现一次，并按原始第一次出现顺序连续排列。

处理进度和日志能反映关键步骤。

失败时日志能给出具体原因。
